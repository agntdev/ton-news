# Ton News roadmap

This roadmap turns the current bounty task list into a practical plan for
content quality, search growth, and product development.

## Current state

Ton News already has the foundation for a community-edited TON publication:

- Next.js app structure with article listing pages
- Submission and moderation flow
- Seeded authentication for local development
- Task-based project plan across platform, content, SEO, community, and
  maintenance work
- Architecture documentation for contributors

## Gaps to address

| Area | Current gap | Why it matters |
| --- | --- | --- |
| Content depth | Few evergreen articles and limited topical coverage | Search engines and readers need useful archives. |
| Editorial consistency | Calendar, guidelines, and review habits need routine use | Consistency keeps quality high as contributors join. |
| Distribution | Social sharing, analytics, and reporting are not complete | Editors need feedback loops for growth. |
| Community trust | Moderation and community policies need visible enforcement | Clear rules reduce spam and promotional content. |
| Storage resilience | TON storage integration still needs production hardening | The publication goal includes decentralized hosting. |

## 30-day priorities

1. Publish the initial evergreen article set for TON basics, wallets,
   validators, staking, and developer tooling.
2. Add SEO metadata, sitemap, robots, and article structured data.
3. Use a weekly content calendar for topic planning and publication tracking.
4. Document moderation decisions for every rejected or revised submission.
5. Add basic analytics for article views, referrers, and search terms.

## 60-day priorities

1. Add social sharing cards and recommended copy for each article.
2. Add comment or feedback collection with moderation safeguards.
3. Create monthly editorial reports covering published articles, search
   performance, submissions, and rejected content categories.
4. Improve mobile performance and accessibility across submission and article
   pages.
5. Pilot TON storage publishing for approved long-form articles.

## 90-day priorities

1. Build a public topic hub for validators, wallets, DeFi, developer tools, and
   ecosystem safety.
2. Add contributor profiles and transparent disclosure fields.
3. Add automated backups for content and moderation records.
4. Create an editorial dashboard for pending, scheduled, and published content.
5. Define a quarterly roadmap review with maintainers and active contributors.

## Success metrics

- At least 10 evergreen articles published.
- At least 3 articles published per week for 4 consecutive weeks.
- Every published article has source links, tags, excerpt, and review notes.
- Sitemap includes all public articles.
- Median article page passes Core Web Vitals targets in production.
- Moderation decisions are documented with a reviewer note.

## Roadmap review cadence

- Weekly: review article pipeline and immediate blockers.
- Monthly: review content performance and update topic priorities.
- Quarterly: revise roadmap, retire stale priorities, and add new feature
  milestones.
